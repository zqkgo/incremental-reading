# 优先级队列（Priority Queue）

## 定义

优先级队列是增量阅读中用于**管理海量学习材料**的核心机制。在 SuperMemo 中，每一个学习元素（主题、摘录、闪卡）都被赋予一个优先级值（0%～100%），所有元素按照优先级排列成一个有序队列。

> "在增量学习中，所有元素都按其优先级决定的顺序组织成序列，该序列即称为优先级队列。"  
> —— SuperMemo Help

---

## 为什么需要优先级

增量阅读允许你导入无限量的材料，但你的时间和认知资源是有限的。**你永远无法处理完所有导入的内容**——这是增量阅读的设计前提，而非缺陷。

优先级队列解决的核心问题：

- 每天只有有限的学习时间
- 导入的材料往往比可处理的多得多
- 不同材料的价值差异悬殊
- 需要确保最重要的内容被优先处理

**核心哲学**：与其焦虑于"没读完"，不如接受"有意识地优先处理最重要的"。

---

## 优先级的表示方式

SuperMemo 中每个元素有一个 **0%～100%** 的优先级：

```
优先级 = 0%    → 最高优先级（最先处理）
优先级 = 100%  → 最低优先级（最后处理，可能永远等不到）
```

注意：数值越小，优先级越高。这是为了直观地类比"名次"（第1名最优秀）。

---

## 优先级队列的工作原理

### 基本机制

```
每天学习时间开始
        ↓
SuperMemo 从队列顶端（最高优先级）开始呈现元素
        ↓
处理当前元素（阅读/提取/复习）
        ↓
移到下一个元素
        ↓
学习时间耗尽 → 停止（剩余低优先级元素延后）
```

优先级越高的元素越早被看到，低优先级的元素可能被推迟数天、数月甚至永远不被处理。

### 与 SRS 调度的交互

优先级与 SRS 间隔共同决定元素何时出现：

- **SRS** 决定"这个元素现在到期了吗？"
- **优先级** 决定"在所有到期元素中，先看哪个？"

如果今天有 100 个到期元素但只有时间处理 50 个，系统按优先级顺序取前 50 个。

---

## 设置优先级的策略

### 手动设置

在导入文章或创建闪卡时，手动指定优先级：

| 场景                     | 建议优先级 |
| ------------------------ | ---------- |
| 明天考试必考内容         | 1%～5%     |
| 当前工作急需的技能       | 5%～15%    |
| 重要但不紧急的长期知识   | 15%～40%   |
| 感兴趣但非必要的背景知识 | 40%～70%   |
| "有空再看"的文章         | 70%～100%  |

### 自动优先级（Priority Bias）

SuperMemo 支持为整个知识分类（Category）或知识树分支设置默认优先级，新导入的元素自动继承该值，减少手动设置的工作量。

---

## 优先级偏差（Priority Bias）

SuperMemo 中的"优先级偏差"功能允许按照主题类别批量调整优先级分布，确保不同领域的知识都能按期望比例出现在学习队列中：

```
例如：
- 工作相关知识：优先级集中在 5%～20%
- 历史/文化知识：优先级集中在 30%～60%
- 娱乐性阅读：优先级集中在 60%～90%
```

---

## 推迟与放弃

增量阅读的哲学中，对低优先级内容"有意识地放弃"不仅被允许，而且被鼓励：

### 推迟（Postpone）

将某元素的下次复习时间推迟，暂时让出队列位置给更重要的内容。

### 放弃（Dismiss/Delete）

如果一篇文章评估后价值不高，可以直接删除或标记为废弃，从队列中永久移除。

**关键心态**：不读完某篇文章不等于失败。信息爆炸时代，有效的知识管理意味着勇于放弃低价值内容，专注高价值内容。

---

## 优先级 vs. 时间压力

| 情景           | 应对方式                                         |
| -------------- | ------------------------------------------------ |
| 学习时间充裕   | 可适当降低重要材料的优先级门槛，接触更广泛的内容 |
| 学习时间紧张   | 提高优先级阈值，只处理最重要的内容               |
| 队列积压严重   | 批量清理低优先级材料（推迟/放弃），重新聚焦      |
| 新加入紧急材料 | 立即赋予高优先级，确保快速进入处理流程           |

---

## 优先级与注意力管理

优先级队列不仅是技术机制，也是**注意力管理哲学**的体现：

- 将有限的认知资源投入最高价值的内容
- 用算法代替意志力来抵制"次要但有趣"内容的干扰
- 建立清晰的知识价值体系，减少决策疲劳
